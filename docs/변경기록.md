### 주요 결정 사항

#### 공통

- **개인용** 사용 목적 확정

- React + Vite + Tailwind + PWA 스택

- AI 페어 프로그래밍(Cursor/Claude Code) 활용

#### 여행 관리 앱

- **모바일 우선** 설계 (폰이 메인 사용 기기)
- 데이터 모델: Trip → Day → Activity 계층 구조
- Activity 타입: 이동/식사/방문지/숙박/기타
- 동기화: **Supabase 자동 동기화**로 결정
- 사진 저장: 원본은 폰 갤러리/Google Photos, 앱엔 썸네일만 (Supabase Storage)
- 지도: Leaflet + OpenStreetMap (Google Maps 아님)
- 요약본 출력: **PNG만** (PDF는 향후 추가 가능)
- 사진 양 기준: 1박2일 약 100장 → 5년치 약 15GB 예상
- AI 커버 이미지: 앱이 프롬프트만 생성, 외부 무료 툴(Bing Image Creator 등) 사용
- AI 커버 이미지를 **Trip 갤러리의 표지로 사용** (시각적 인덱스)
- AI 이미지 스타일 4종: 수채화 / 지브리풍 / 폴라로이드 콜라주 / 미니멀 플랫
- 입력 모드 분리: 여행 중 모드 (간단) / 정리 모드 (상세)

## 2026-06-30 - [추가·결정] 데이터 모델 대폭 확장, 검색 기능 도입, 디자인 시스템 확정

### 프로젝트

- 여행 관리 앱

### 추가된 요구사항

#### Trip 확장

- **나라 유형**: 국내/해외 구분, 해외 시 국가명 입력
- **지역**: 예 "강원도", "오사카"
- **경로**: 출발지·도착지 (선택), 카드에는 도착지만 표시 ("서울 → 강릉")
- **카테고리**: 바다·산·도시·맛집·문화 (다중 선택 허용, 최대 2개 카드 표시)

#### Day 자동 파생 + 감상 정보

- 명시적 Day 테이블 없이 startDate ~ endDate로 UI 생성
- **dayNotes**: 날짜를 키로 감상 정보 저장
  - wakeTime, sleepTime (선택)
  - weather (선택)
  - mood (컨디션 1-5, ◆ 다이아 5개로 표시)
  - highlight (한 줄, 카드에 표시)
  - journal (자유 텍스트, 상세에서만)
- 모든 필드 선택 사항 (안 채워도 됨)
- Day 카드에 자동 요약: "방문 N곳 · 지출 · 사진 N장"

#### Activity 확장

- **위치**: 주소 텍스트 + GPS 좌표 (선택)
  - 여행 중: 현재 위치 자동 (Geolocation API)
  - 여행 후: 주소 텍스트 입력
- **음식 종류**: 자유 텍스트 + 자동완성 (식사 타입일 때)
- **재방문 의사**: yes/no/maybe (선택)
- **즐겨찾기**: boolean (별 또는 하트 아이콘)

#### 검색 기능 (킬러 기능)

- 홈 우상단 돋보기 → 검색 페이지
- **통합 검색 + 필터** 하이브리드
- 검색어 자유 입력 (예: "강원도 칼국수")
- 필터: 타입(식당·방문지) · 평점 · 재방문 의사
- 지역 매칭: Trip 지역 필드 + Activity 주소 텍스트 둘 다
- 검색 대상: 모든 Activity, 필터로 좁히기
- 결과 카드 클릭 → 식당(장소) 전용 페이지

#### 식당(장소) 전용 페이지 (신규)

- 여러 번 방문한 경우 이력 모아보기
- 상호명·주소·태그·평균 평점·방문 횟수 표시
- 즐겨찾기 토글 (별 또는 하트)
- 새 방문 추가 기능
- 공유 기능 (링크 복사)
- 방문 이력 카드 스택 (최근순): 날짜·시간·여행·평점·메모·가격

### 확정된 디자인 방향 (초기 결정에서 변경)

- **배경 톤**: 순백(#FFFFFF) → **크림(#FAF9F5)** 로 변경
- **주 색상**: 선명한 파랑(#3B82F6) → **청회색 (#1E2A38, #3A4A5C, #6B8AA8)** 로 변경
- **분위기**: 밝은 감성 → **감성 저채도**
- **액센트**: 파랑만 → **갈색빛 (#B08D5C)** 별점·즐겨찾기·하이라이트에만
- **별점**: ★ (별) → **◆ (다이아)** 5개
- **컨디션**: 별점과 동일한 ◆ 5개
- **즐겨찾기**: 별(ti-star-filled) 또는 하트(ti-heart-filled) 사용
- **폰트**: 산세리프만 (한글 위주, Pretendard 우선)
- **날짜**: 리스트 영문(JUN 28), 상세 숫자(6.28) 혼합
- **아이콘**: Tabler Icons Outline
- **하단 탭바**: 홈 · 지도 · 통계 · 나 (+중앙 FAB)

### 변경 이유

1. **감성 저채도로 방향 변경**
   - 4가지 시안 비교 후 결정
   - 오래 봐도 안 질리는 톤
   - "여행 하나하나가 소중한 기억"이라는 톤
   - 실용 정보(통계·지출)는 유지

2. **검색·식당 페이지 도입**
   - 사용자 실제 경험: "예전 갔던 식당 다시 가고 싶은데 어딘지 기억 안남"
   - 여행 앱의 킬러 실용 기능
   - 개인용 맛집 DB 역할

3. **Day 자동 파생**
   - 명시적 Day 테이블은 관리 복잡도 큼
   - startDate/endDate만 있으면 Day 자동 생성 가능
   - dayNotes로 감상 정보만 별도 저장 → 데이터 구조 단순

4. **영수증 저장 안 함**
   - 저장 목적 아닌 입력 도우미로만 사용
   - Tesseract.js 한글 OCR 정확도 낮음
   - 폰 기본 OCR(Google Lens, iOS 라이브 텍스트) 활용 안내로 대체

5. **사진 저장 전략**
   - 원본은 폰 갤러리에 이미 백업됨
   - 앱에는 썸네일(300KB) + 중간 크기(1MB)만 저장
   - HEIC 자동 JPG 변환 (heic2any)
