## 2026-07-05 - [추가·결정] 폰트 프리셋 시스템 도입, Pretendard 제거

### 프로젝트

- 여행 관리 앱

### 변경 내용

폰트를 사용자가 마이페이지에서 선택할 수 있는 프리셋 시스템으로 확장. 기존 Pretendard 단일 폰트에서 5개 프리셋(제목/본문 조합)으로 변경.

#### 폰트 프리셋 5개

| key           | 라벨       | 제목 폰트       | 본문 폰트    | 무드                                 |
| ------------- | ---------- | --------------- | ------------ | ------------------------------------ |
| `editorial`   | 에디토리얼 | Maru Buri       | Gowun Dodum  | 감성 명조 + 부드러운 산세리프 (기본) |
| `soft`        | 부드러움   | Gowun Dodum     | Gowun Dodum  | 통일된 다정한 산세리프               |
| `modern`      | 모던       | Wanted Sans     | Wanted Sans  | 깔끔한 모던 산세리프                 |
| `classic`     | 감성 명조  | Gowun Batang    | Gowun Batang | 클래식 명조 통일                     |
| `handwriting` | 손글씨     | Ownglyph 미인체 | Gowun Dodum  | 실제 손글씨 제목 (개성적)            |

#### 신규 파일

- `src/theme/fonts.js` — 5개 프리셋 정의 + helper (allFonts, defaultFont, getFont)

#### 수정 파일

- `src/theme/ThemeContext.jsx` — fontKey 상태 추가, `applyFont` 함수 (CSS 변수 `--font-heading`, `--font-body` 주입), 별도 localStorage 저장
- `src/theme/useTheme.js` — JSDoc에 font 관련 필드 추가 (font, fontKey, setFont, allFonts)
- `src/index.css` — `@theme inline` 에 `--font-heading`, `--font-body` 매핑 추가, `html/body` 에 `font-family: var(--font-body)` 지정
- `index.html` — Pretendard `<link>` 제거, 5개 폰트 CDN 로드 추가 (Google Fonts + jsDelivr + 눈누)
- **`font-heading` 유틸 적용 지점 9개**:
  - UI 컴포넌트: `Section` `<h2>`, `PageHeader` `<h1>`
  - 페이지 헤더: `HomePage`, `FavoritesPage`, `StatsPage`, `ProfilePage`, `TripDetailPage` (히어로 h1)
  - 카드 제목: `TripCard` (`<h3>`), `ActivityItem` (`<h4>`)
- `src/pages/ProfilePage.jsx` — `FontSection` + `FontCard` 서브컴포넌트 추가, 테마 섹션과 데이터 관리 섹션 사이 배치

### 배경·이유

1. **Pretendard "안 이쁘다" 사용자 피드백**
   - 리팩토링 완료 후 실사용 테스트 시점에 명확히 나타난 감상
   - 여행 앱의 감성 저채도 톤과 매치 안 됨
   - 산세리프 하나로는 다양한 무드 표현 어려움

2. **테마 시스템 인프라 재활용 가능**
   - 이미 있는 CSS 변수 주입, localStorage, 마이페이지 카드 UI 를 그대로 확장
   - 별도 FontContext 만들지 않고 ThemeContext 통합
   - 예상 시간 3-5시간 → 실제 반나절 내 완료

3. **프리셋 방식이 개별 선택보다 유리**
   - 사용자가 폰트 조합을 고민할 필요 없음
   - 실사용 중 실험이 쉬움 (원터치 전환)
   - 새 조합 추가는 개발자 결정 사항으로 남김

### 확정 결정사항

#### 폰트 정책

- **폰트 이름 규칙**: 제목 = heading, 본문 = body
- **적용 방식**: `font-heading` 유틸을 명시적 적용, 나머지는 `html/body` 기본값(`font-body`)을 상속
- **폴백**: 시스템 폰트 (`-apple-system, Malgun Gothic` 등). 웹 폰트 로드 실패해도 앱 정상 동작
- **폼 소제목은 body 유지**: ActivityForm "새 일정 추가", TripForm "새 여행 추가" 등 UI 라벨 성격이라 heading 미적용

#### localStorage 정책

- 폰트는 테마와 별도 저장: `travel-app:font = 'editorial'` (문자열 key)
- **데이터 초기화 대상 아님** (마이페이지의 "초기화" 는 `trips` 만 삭제)

#### 폰트 로드 방식

- **CDN 방식** (개별 다운로드 X): Google Fonts + jsDelivr + 눈누
- 5개 폰트 모두 초기 로드 (트래픽 개인용이라 부담 없음)
- 각 폰트 라이선스: 모두 OFL 또는 상업 무료

### 시각적 변화

- 앱 전체 폰트 톤이 크게 바뀜 (Pretendard 산세리프 → Maru Buri 명조 + Gowun Dodum 조합이 기본)
- 여행 앱의 "감성 저채도" 톤과 훨씬 잘 어울림
- 카드 제목, 페이지 헤더가 명조로 렌더되어 다이어리/에세이 느낌 강화
- 본문/라벨/폼은 부드러운 산세리프로 가독성 유지

### 영향 범위

- `docs/디자인_시스템.md` V3.1 로 갱신 (섹션 3 전면 재작성, 16 CSS 변수 추가, 20 fonts.js 참조 추가, 12.7 폰트 섹션 추가)
- `docs/변경기록.md` 본 항목 추가
- 신규 파일 1개: `src/theme/fonts.js`
- 수정 파일 12개: ThemeContext, useTheme, index.html, index.css, Section, PageHeader, TripCard, ActivityItem, TripDetailPage, HomePage, FavoritesPage, StatsPage, ProfilePage

### 후속 작업 (선택)

- [ ] 실사용 후 기본 프리셋을 `editorial` 이 아닌 다른 것으로 변경할지 검토
- [ ] Ownglyph 미인체 CDN URL 안정성 확인 (눈누 CDN은 변경 가능성)
- [ ] 폰트별 성능 측정 (초기 로드 시간)
- [ ] 다크 테마 + 손글씨 프리셋 조합 시 가독성 확인
- [ ] 폼 필드 (Input, Textarea) 안의 텍스트가 body 폰트로 잘 렌더되는지 확인

## 2026-07-03 - [추가·결정] 테마 시스템 도입, UI 컴포넌트 라이브러리 구축, 전면 리팩토링

### 프로젝트

- 여행 관리 앱

### 변경 내용

Phase 1~5로 나눠서 진행한 전면 리팩토링 완료. 인라인 style 하드코딩 방식에서 테마 시스템 + UI 컴포넌트 기반으로 전환.

#### Phase 1 - 디자인 토큰 + 8개 테마 프리셋

- `src/theme/tokens.js` — 공통 토큰 (spacing, radius, shadow, typography, transition, rating)
- `src/theme/themes.js` — 8개 테마 프리셋 (라이트 5, 다크 3) + helper
- **12개 semantic 색상 토큰**으로 통일 (bg, surface, surfaceAlt, text, textMuted, textSubtle, border, borderStrong, accent, accentFg, danger, success)
- 평점 색은 골드(`#D4A017`) 고정, 테마 무관

#### Phase 2 - ThemeProvider

- `src/theme/ThemeContext.jsx` — Context + Provider + CSS 변수 자동 주입
- `src/theme/useTheme.js` — 소비용 훅
- localStorage 저장 (`travel-app:theme` = `{ key, autoDetect }`)
- 시스템 다크모드 자동 감지 + 수동 오버라이드
- `<main.jsx>` 에서 `<ThemeProvider>` 로 앱 감쌈
- `index.css` 에 `@theme inline` 매핑 추가 (Tailwind v4 유틸리티 자동 생성)

#### Phase 3 - 공통 UI 컴포넌트 12개

`src/components/ui/` 에 재사용 컴포넌트 라이브러리 구축:

- **Card** — surface 컨테이너 (padding: none/sm/md/lg)
- **Button** — 4개 variant (primary/secondary/ghost/danger), 3개 size, leftIcon/rightIcon/loading
- **Chip** — 태그 · 필터 · 선택 상태 (default/selected/outline)
- **Input · Textarea · Select** — 에러 상태, forwardRef, focus 자동 처리
- **Label** — required 표시 지원
- **Badge** — 카운트 · 상태 (5개 variant, dot 모드)
- **Section** — 페이지 내부 그룹핑 (title + description + action)
- **PageHeader** — 뒤로가기 + 액션 슬롯
- **Rating** — **◆ 다이아몬드, 0.5 단위 지원** (편집 모드에서 좌/우 반쪽 클릭)
- **Switch** — iOS 스타일 토글
- `ui/index.js` — barrel export

#### Phase 4 - 기존 컴포넌트/페이지 11개 리팩토링

- **BottomTabBar** — `TabLink` 서브컴포넌트 추출, FAB → accent 색
- **TripCard** — Rating 컴포넌트 적용, `renderStars` 삭제, 아이콘 이모지 → Tabler
- **TripList** — Section + Card 사용, 빈 상태 처리
- **ActivityForm** — 인라인 style 100% 제거, Input/Textarea/Chip/Rating 대체
- **ActivityItem** — `MiniBadge` 서브컴포넌트, Rating 적용, 이모지 → Tabler
- **TripForm** — Select 컴포넌트, Chip 토글 (국내/해외, 카테고리)
- **HomePage** — 헤더 테마 유틸화, 안 쓰이는 import 정리
- **TripDetailPage** — 히어로 커버, `StatCell` 서브컴포넌트, Rating + Textarea 통합
- **FavoritesPage · StatsPage · ProfilePage** (기존 placeholder) — 테마 유틸 적용

#### Phase 5 - 마이페이지 확장

- **ProfilePage** placeholder → 실제 설정 페이지로 확장
- **테마 섹션**: 자동 감지 Switch + 2열 그리드 카드 (라이트 5, 다크 3)
  - `ThemeCard`: 팔레트 원 4개 (accent, bg, surface, text) + 이름 + 선택 시 체크
  - `PaletteCircle`: 살짝 겹친 원, `ring-2 ring-surface` 로 구분
- **데이터 관리 섹션**: 내보내기 (JSON 다운로드) · 가져오기 (JSON 업로드) · 초기화 (이중 confirm)
- **정보 섹션**: 앱 버전
- `DataMenuItem` 서브컴포넌트로 리스트 항목 통일

#### 추가 - 테마별 히어로 그라데이션

- 초기 리팩토링에서 히어로 커버/썸네일 그라데이션을 하드코딩 유지했으나 테마 전환 시 어색함 발견
- 8개 테마 각각에 `heroFrom`, `heroTo` 색상 추가
- CSS 변수 `--color-hero-from`, `--color-hero-to` 로 노출
- `TripCard` (썸네일) 과 `TripDetailPage` (히어로) 공용
- Tailwind 유틸 `from-hero-from to-hero-to` 로 사용

### 배경·이유

1. **테마 시스템 필요성**
   - 감성 저채도 크림 톤 하나로는 사용자 취향/상황(주간·야간) 대응 어려움
   - 다크 모드 지원은 모바일 앱의 표준 기대 사항
   - 코드에 hex 하드코딩된 상태에서는 확장 불가

2. **UI 컴포넌트 라이브러리**
   - 인라인 style + `onFocus/onBlur` 이벤트 등 유사 코드가 파일마다 반복
   - `ActivityForm` 원본 690줄 → 리팩토링 후 520줄 (25% 감소, 실질 JSX는 훨씬 적음)
   - 평점 렌더 함수(`renderStars`)가 여러 파일에 중복 → Rating 컴포넌트로 통합, 0.5 지원까지 획득

3. **마이페이지 확장**
   - 테마 선택 UI 는 테마 시스템의 사용자 접점으로 필수
   - 데이터 관리 (백업/복원/초기화) 는 localStorage 기반 앱의 실용 기능

### 확정된 결정사항

#### 색상/토큰

- CSS 변수 네이밍 컨벤션: **kebab-case** (`--color-bg`, `--color-surface-alt`)
- semantic 토큰 12개로 통일 (이전 문서의 세분화된 15개 → 정리)
- 평점 색 골드 고정, 다른 테마와 무관
- 히어로 그라데이션 = 테마별 (2색), TripCard 썸네일과 상세 히어로 공용

#### 컴포넌트 정책

- **하드코딩 hex 금지** (오버레이 뱃지, 그라데이션 대체 이미지 자리만 예외)
- **인라인 `style` 금지** (Tailwind arbitrary value로 대체 or 컴포넌트 사용)
- **아이콘 `color` prop 금지** — 부모 `text-*` 유틸로 CSS 상속
- **이모지 아이콘 → Tabler 통일** (📍 → IconMapPin, ✕ → IconX 등)
- **폼 필드의 `onFocus/onBlur` 인라인 이벤트 금지** — Input 컴포넌트가 자동 처리

#### Rating 세부 스펙

- 0.5 단위 지원 (반쪽 다이아 표시)
- 편집 모드: 각 다이아를 좌/우 반쪽으로 나눠 클릭 (좌 = +0.5, 우 = +1.0)
- 같은 값 재클릭 시 0 으로 해제 (일반적 rating UX)
- 호버 미리보기 지원

#### 데이터 관리 (마이페이지)

- 백업 파일 형식: `{ version: 1, exportedAt, trips }`
- 파일명: `travel-app-backup-YYYY-MM-DD.json`
- 초기화: 이중 confirm (실수 방지), 성공 시 자동 새로고침
- 초기화 대상: `trips` 만. 테마 설정 (`travel-app:theme`) 은 보존

### 시각적 변화

원본 대비 사용자가 인지할 수 있는 변화:

- **크림 테마 FAB 색이 갈색빛(accent)로 변경** — 원본은 짙은 청회색(text 색과 동일). 통일된 accent 개념으로 이동

- **평점 크기 근사치 유지** — 원본 text-2xl (24px) → Rating size lg (26px)
- **아이콘/버튼에 hover 상태 추가** — 원본은 대부분 무반응

## 2026-06-30 - [추가·결정] 데이터 모델 대폭 확장, 검색 기능 도입, 디자인 시스템 확정

### 추가된 요구사항

#### Trip 확장

- **나라 유형**: 국내/해외 구분, 해외 시 국가명 입력
- **지역**: 예 "강원도", "오사카"
- **경로**: 출발지·도착지 (선택), 카드에는 도착지만 표시 ("서울 → 강릉")
- **카테고리**: 바다·산·도시·맛집·문화 (다중 선택 허용, 최대 2개 카드 표시)

#### Day 자동 파생 + 감상 정보

- 명시적 Day 테이블 없이 startDate ~ endDate로 UI 생성
- **dayNotes**: 날짜를 키로 감상 정보 저장
  - wakeTime, sleepTime (선택)
  - weather (선택)
  - mood (컨디션 1-5, ◆ 다이아 5개로 표시)
  - highlight (한 줄, 카드에 표시)
  - journal (자유 텍스트, 상세에서만)
- 모든 필드 선택 사항 (안 채워도 됨)
- Day 카드에 자동 요약: "방문 N곳 · 지출 · 사진 N장"

#### Activity 확장

- **위치**: 주소 텍스트 + GPS 좌표 (선택)
  - 여행 중: 현재 위치 자동 (Geolocation API)
  - 여행 후: 주소 텍스트 입력
- **음식 종류**: 자유 텍스트 + 자동완성 (식사 타입일 때)
- **재방문 의사**: yes/no/maybe (선택)
- **즐겨찾기**: boolean (별 또는 하트 아이콘)

#### 검색 기능 (킬러 기능)

- 홈 우상단 돋보기 → 검색 페이지
- **통합 검색 + 필터** 하이브리드
- 검색어 자유 입력 (예: "강원도 칼국수")
- 필터: 타입(식당·방문지) · 평점 · 재방문 의사
- 지역 매칭: Trip 지역 필드 + Activity 주소 텍스트 둘 다
- 검색 대상: 모든 Activity, 필터로 좁히기
- 결과 카드 클릭 → 식당(장소) 전용 페이지

#### 식당(장소) 전용 페이지 (신규)

- 여러 번 방문한 경우 이력 모아보기

- 방문 이력 카드 스택 (최근순): 날짜·시간·여행·평점·메모·가격
