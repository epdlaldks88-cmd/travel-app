# 여행 관리 앱 기획서

> 개인용 React 기반 여행 기록·회상 앱
> 최종 업데이트: 2026-06-30

---

## 1. 프로젝트 개요

### 1.1 목적
여행을 기록하고, 사후에 시각적으로 회상할 수 있는 개인용 앱.
주로 폰에서 사용하며, AI 생성 커버 이미지로 갤러리처럼 추억을 정리한다.

### 1.2 사용자
- 개인 1명 (본인)
- 주 사용 기기: 스마트폰
- 보조 기기: PC (사후 정리용)

### 1.3 핵심 가치
1. **빠른 기록**: 여행 중 폰으로 간단히 입력
2. **시각적 회상**: AI 커버 이미지로 한눈에 알아보기
3. **0원 운영**: 무료 서비스만 사용

---

## 2. 핵심 기능

### 2.1 데이터 모델

```
Trip (여행)
 ├─ 제목, 시작·종료일, 동행자, 총 비용(자동 합산)
 ├─ 전체 평점, 한줄평
 ├─ AI 커버 이미지
 │
 └─ Day (일자)
     ├─ 날짜, 그날의 메모
     │
     └─ Activity (활동)
         ├─ 타입: 이동 / 식사 / 방문지 / 숙박 / 기타
         ├─ 공통: 시작·종료 시각, 위치(GPS), 비용, 메모, 사진들, 평점
         │
         ├─ 이동: 교통수단, 출발지, 도착지, 소요시간, 거리, 주유비/통행료
         ├─ 식사: 식당명, 음식 종류, 메뉴, 맛 평점, 가성비, 재방문 의사
         ├─ 방문지: 장소명, 카테고리, 입장료, 머문 시간, 재방문 의사
         └─ 숙박: 숙소명, 체크인·아웃, 1박 가격, 평점

Photo: Activity에 속함, 파일 경로, 촬영 시각, GPS, 캡션
Companion: 동행자 (재사용 가능, 별도 테이블)
```

### 2.2 입력 모드

#### 여행 중 모드 (폰)
- 큰 버튼, 최소 입력
- 사진 + 위치 + 한줄 메모만 빠르게 저장
- 현재 위치 원클릭 입력 (Geolocation API)
- 오프라인 입력 지원

#### 정리 모드 (폰 또는 PC)
- 모든 필드 보강
- 사진 일괄 드롭존 → EXIF로 자동 분류
- 타임라인 자동 초안 생성

### 2.3 뷰

- **갤러리 뷰**: AI 커버 이미지 + 제목/날짜/동행자 카드 격자
- **지도 뷰**: 경로 라인 + 핀
- **타임라인 뷰**: 일자별 시간순 표시
- **통계 대시보드**: 연도별 횟수, 총 지출, 자주 간 지역, 평점 TOP

### 2.4 요약본 및 AI 커버

**정보 요약본 (앱이 직접 생성)**
- React 템플릿 + html2canvas → PNG 출력
- 구성: 제목/날짜/동행자, 미니 지도, 일정 타임라인, 식당 베스트, 하이라이트 사진, 총 지출, 한줄 회고

**AI 커버 이미지 (외부 툴 사용)**
- 앱이 여행 데이터 기반 프롬프트 자동 생성
- 스타일 옵션 4종: 수채화 / 지브리풍 / 폴라로이드 콜라주 / 미니멀 플랫
- 프롬프트 복사 버튼 + 추천 무료 툴 안내 (Bing Image Creator 등)
- 사용자가 외부에서 생성한 이미지를 업로드 → Trip 커버로 저장
- 가로 비율 16:9 또는 3:2 (프롬프트에 "landscape orientation" 명시)

### 2.5 검색·필터
- 조건 검색: "평점 4점 이상 일식집", "작년 강원도에서 먹은 것" 등
- 재방문 의사 있는 곳 리스트

## 3. 기술 스택

| 영역 | 기술 | 비용 |
|---|---|---|
| Frontend | React + Vite + Tailwind | 0원 |
| 배포 형태 | PWA (폰 홈화면 설치) | 0원 |
| DB | Supabase (PostgreSQL) | 0원 (Free tier) |
| 인증 | Supabase Auth (이메일/구글) | 0원 |
| 로컬 캐시 | Dexie.js (IndexedDB) | 0원 |
| 사진 저장 | Supabase Storage (썸네일) | 0원 (Free tier 1GB) |
| 원본 사진 | 폰 갤러리 + Google Photos/iCloud | 0원 |
| 지도 | Leaflet + OpenStreetMap | 0원 |
| EXIF 추출 | exifr | 0원 |
| 요약본 생성 | html2canvas | 0원 |
| 호스팅 | Vercel 또는 Netlify | 0원 |
| **총 운영비** | | **0원** |

### 3.1 사진 저장 전략
- 1박2일 약 100장 기준 → 5년치 약 15GB
- **원본은 폰 갤러리/Google Photos에 남기고, 앱에는 썸네일(300KB)만 저장**
- 썸네일 5년치 약 1.5GB → Supabase Free tier로 충분

### 3.2 동기화 전략
- 폰 메인 + PC 정리 → Supabase 자동 동기화 사용
- 오프라인 입력 시 IndexedDB 임시 저장 → 네트워크 복구 시 자동 sync

## 4. UI/UX 원칙

- **모바일 우선 (Mobile-first)**
- 터치 친화: 큰 버튼, 칩 형태 선택, 단축 입력
- 평점은 별 탭, 재방문 의사는 Yes/No/Maybe 버튼
- 카메라 직접 호출: `<input type="file" accept="image/*" capture="environment">`
- 입력 폼은 키보드 Tab으로 빠르게 이동

## 5. MVP 로드맵 (수정판)

> ⚠️ **React 익숙하지 않음을 반영한 단계적 접근**

### Step 0: React 워밍업 (1~2주)
- 미니 프로젝트로 React 기본기 잡기
- 예: "Trip 하나 등록 + 리스트 표시" (DB 없이 메모리만 → localStorage)
- useState, useEffect, 컴포넌트 분리, 폼 핸들링 익히기

### Step 1: 최소 기능 여행 앱 (2~3주)
- **로컬에서만 작동** (Supabase X, PWA X)
- localStorage 또는 Dexie.js
- Trip 등록 / 리스트 / 상세 / 사진 1장 / 삭제
- 모바일 반응형
- 폰 브라우저에서 접속 가능

### Step 2: 실제 사용 (2~4주)
- **한두 번 실제 여행에서 써보기**
- 불편한 점, 진짜 필요한 기능 발견
- 머리로 설계한 것과 실제 사용의 갭 확인

### Step 3: 필요한 것만 점진적 추가
- Step 2에서 진짜 필요하다고 느낀 것부터
- 예시 순서:
  1. Supabase 동기화
  2. Leaflet 지도
  3. EXIF 자동 분류
  4. 요약본 + AI 프롬프트 생성
  5. 통계 대시보드
  6. PWA 설치 최적화

### 핵심 원칙
- **처음부터 다 만들지 않는다**
- **작게 만들고 빨리 써보고 고치는 사이클**
- AI 페어 프로그래밍(Cursor/Claude Code) 적극 활용하되, 코드는 한 줄씩 이해
