# 여행 관리 앱 기획서

> 개인용 React 기반 여행 기록·회상 앱
> 최종 업데이트: 2026-06-30 (V2)

---

## 1. 프로젝트 개요

### 1.1 목적

여행을 기록하고, 사후에 시각적으로 회상할 수 있는 개인용 앱.
주로 폰에서 사용하며, AI 생성 커버 이미지로 갤러리처럼 추억을 정리한다.
**과거 방문한 식당·장소를 검색하여 재방문할 수 있게 하는 것**이 핵심 실용 기능.

### 1.2 사용자

- 개인 1명 (본인)
- 주 사용 기기: 스마트폰
- 보조 기기: PC (사후 정리용)

### 1.3 핵심 가치

1. **빠른 기록**: 여행 중 폰으로 간단히 입력
2. **시각적 회상**: AI 커버 이미지로 한눈에 알아보기
3. **재방문 지원**: 지역·음식·평점으로 검색 → 과거 맛집 찾기
4. **0원 운영**: 무료 서비스만 사용

## 2. 핵심 기능

### 2.1 데이터 모델 (최종)

```
Trip (여행)
├─ id
├─ 제목
├─ 나라 유형: "국내" | "해외"
├─ 국가명 (해외일 때만, 예: "일본")
├─ 지역 (예: "강원도", "오사카")
├─ 출발지 (선택)
├─ 도착지 (선택)
├─ 시작일, 종료일
├─ 동행자 (배열)
├─ 카테고리 (배열: ["바다", "도시"])
├─ 전체 평점 (1-5)
├─ 한줄 회고
├─ AI 커버 이미지 (사용자 업로드)
├─ 총 지출 (자동 합산)
│
├─ dayNotes: {
│    "2026-06-28": {
│      wakeTime, sleepTime, weather, mood (1-5),
│      highlight (한 줄), journal (자유 텍스트)
│    }
│  }
│
└─ Activity[] (일정)
    ├─ id
    ├─ 타입: "이동" | "식사" | "방문지" | "숙박" | "기타"
    ├─ 날짜
    ├─ 시간
    ├─ 이름 (예: "옛맛 칼국수", "안목해변")
    ├─ 위치: { 주소, 위도, 경도 } (선택)
    ├─ 음식 종류 (식사일 때, 자유 텍스트 + 자동완성)
    ├─ 재방문 의사: "yes" | "no" | "maybe" | null
    ├─ 즐겨찾기: boolean
    ├─ 비용
    ├─ 평점 (1-5)
    ├─ 메모
    └─ 사진들 [{ 썸네일 URL, 중간 크기 URL }]
```

**Day는 자동 파생**: 명시적 Day 테이블 없이 Trip의 startDate ~ endDate로 UI 생성. dayNotes는 날짜를 키로 저장.

### 2.2 입력 모드

#### 여행 중 모드 (폰)

- 큰 버튼, 최소 입력
- 사진 + 위치 + 한줄 메모만 빠르게 저장
- 현재 위치 원클릭 입력 (Geolocation API)
- 오프라인 입력 지원

#### 정리 모드 (폰 또는 PC)

- 모든 필드 보강
- 사진 일괄 드롭존 → EXIF로 자동 분류
- Day 감상문 작성
- 재방문 의사·즐겨찾기 표시

### 2.3 뷰

- **갤러리 뷰**: AI 커버 이미지 + 제목/날짜/동행자 카드 격자
- **지도 뷰**: 경로 라인 + 핀
- **타임라인 뷰**: 일자별 시간순 표시 (Day 카드 스택)
- **통계 대시보드**: 연도별 횟수, 총 지출, 자주 간 지역, 평점 TOP

### 2.4 검색 기능 (핵심)

#### 진입점

- 홈 우상단 돋보기 아이콘

#### 검색 방식

**통합 검색 + 필터** (하이브리드)

- 검색어 자유 입력 (예: "강원도 칼국수")
- 필터로 좁히기: 타입(식당·방문지) · 평점 · 재방문 의사

#### 검색 대상

- 모든 Activity (기본)
- 필터로 특정 타입만 좁히기

#### 지역 매칭

- Trip의 지역 필드
- Activity의 위치 주소 텍스트
- 둘 다 검색

#### 결과 UI

- 카드 리스트: 상호명·지역·음식종류·평점·가격·방문일·메모 한줄
- 클릭 시 **식당(또는 장소) 전용 페이지**로 이동

### 2.5 식당(장소) 전용 페이지

여러 번 방문한 경우 이력을 모아 보여줌.

**표시 내용**:

- 상호명, 주소, 태그(음식 종류)
- 평균 평점 (여러 방문의 평균)
- 즐겨찾기 표시 (별 · 하트 아이콘)
- 방문 횟수
- 방문 이력 리스트 (최근순): 날짜·시간·해당 여행·평점·메모·가격·사진

**기능**:

- 즐겨찾기 토글 (다시 가고 싶은 곳)
- 새 방문 추가 (여기서 바로 다음 방문 기록)
- 공유 (링크 복사)

### 2.6 요약본 및 AI 커버

**정보 요약본 (앱이 직접 생성)**

- React 템플릿 + html2canvas → PNG 출력
- 구성: 제목/날짜/동행자, 미니 지도, 일정 타임라인, 식당 베스트, 하이라이트 사진, 총 지출, 한줄 회고

**AI 커버 이미지 (외부 툴 사용)**

- 앱이 여행 데이터 기반 프롬프트 자동 생성
- 스타일 옵션 4종: 수채화 / 지브리풍 / 폴라로이드 콜라주 / 미니멀 플랫
- 프롬프트 복사 버튼 + 추천 무료 툴 안내 (Bing Image Creator 등)
- 사용자가 외부에서 생성한 이미지를 업로드 → Trip 커버로 저장
- 가로 비율 16:9 또는 3:2

### 2.7 사진 관리

**저장 전략**

- 원본은 폰 갤러리·Google Photos·iCloud 유지
- 앱에는 **썸네일 (300KB, 800px)** + **중간 크기 (1MB, 1600px)** 저장
- 5년치 약 1.5GB → Supabase Free tier 여유

**업로드**

- Trip 상세 페이지에서 일괄 업로드
- 카메라 촬영 or 갤러리 선택
- EXIF 자동 추출 → Day별 자동 그룹핑 → Activity별 자동 매칭
- HEIC 자동 JPG 변환 (heic2any)

**Activity 관리**

- 사진은 Activity에만 속함
- Trip 앨범 = Activity 사진들 모아 보기
- 자동 매칭 잘못됐으면 다른 Activity로 옮기기 가능

### 2.8 영수증

**저장 안 함.** OCR 도구로도 안 씀. 대신:

- Activity 폼에 "Google Lens / iOS 라이브 텍스트 사용 팁" 안내

## 4. UI/UX 원칙

- **모바일 우선 (Mobile-first)**
- 터치 친화: 큰 버튼, 칩 형태 선택
- 평점·컨디션은 별 탭 (◆ 다이아 5개)
- 재방문 의사는 Yes/No/Maybe 버튼
- 즐겨찾기는 별 · 하트 아이콘 토글
- 카메라 직접 호출: input capture="environment"
- 크림 배경 + 청회색 톤 (감성 저채도)
- 자세한 규칙은 `디자인_시스템.md` 참조
