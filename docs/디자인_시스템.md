# 여행 관리 앱 · 디자인 시스템

> 화이트 + 청회색 감성 저채도 · 크림 배경 · 실용 정보 병행
> 최종 업데이트: 2026-06-30

---

## 1. 디자인 원칙

1. **차분한 감성** — 조용하고 정돈된 톤. 과한 장식·화려한 색 배제
2. **실용 정보 병행** — 감성만이 아닌 통계·지출·평점 등 정보도 명확히
3. **모바일 우선** — 폰에서 편하게 쓸 수 있는 크기·간격
4. **일관성** — 같은 요소는 항상 같은 모양·색
5. **여백은 넉넉하게** — 밀도보다 여유

### 2.5 그라디언트 (커버 이미지 자리)

사진 없을 때 커버 이미지 placeholder로 사용.

```css
/* 기본 (일반 여행) */
--gradient-cover: linear-gradient(135deg, #6b8aa8 0%, #3a4a5c 100%);

/* 밝은 (리스트 카드) */
--gradient-cover-light: linear-gradient(135deg, #a8c0d6 0%, #6b8aa8 100%);

/* 아주 밝은 (작은 카드) */
--gradient-cover-lighter: linear-gradient(135deg, #c4d4e4 0%, #a8c0d6 100%);
```

### 2.6 색상 사용 원칙

- **한 화면에 색은 3~4가지만** (배경·텍스트·태그·강조)
- **파랑 계열은 청회색 톤**만 사용, 선명한 파랑(`#3B82F6` 등) 금지
- **갈색빛(`#B08D5C`)은 별점에만** 사용, 다른 곳에 남용 금지
- **버튼·CTA는 진한 청회색** (`#1E2A38`) 사용
- **경고·에러 색**: 나중에 필요 시 추가 (지금은 사용처 없음)

## 3. 타이포그래피

### 3.1 폰트 패밀리

**한글 위주이므로 산세리프만 사용.** 세리프는 사용 안 함.

```css
--font-sans:
  -apple-system, BlinkMacSystemFont, "Pretendard", "Apple SD Gothic Neo",
  "Malgun Gothic", "맑은 고딕", system-ui, sans-serif;
```

우선순위:

1. **Pretendard** (한글 최적화 오픈소스 폰트, 있으면 사용)
2. **Apple SD Gothic Neo** (iOS)
3. **Malgun Gothic / 맑은 고딕** (Windows)
4. **system-ui** (기타)

> **참고**: Pretendard는 나중에 CDN으로 로드하거나 npm 패키지로 추가 (선택 사항). 지금은 시스템 폰트만 사용해도 충분함.

### 3.2 폰트 크기 (모바일 기준)

| 이름             | 크기 | 용도                                | Tailwind      |
| ---------------- | ---- | ----------------------------------- | ------------- |
| `--text-hero`    | 24px | 화면 히어로 제목 (상세 페이지 커버) | `text-2xl`    |
| `--text-title`   | 18px | 페이지 제목 (홈 헤더)               | `text-lg`     |
| `--text-heading` | 16px | 섹션 제목 (여행 카드 제목)          | `text-base`   |
| `--text-body`    | 14px | 본문 · 리스트 항목                  | `text-sm`     |
| `--text-caption` | 12px | 부가 정보 · 날짜 · 라벨             | `text-xs`     |
| `--text-micro`   | 10px | 태그 · 뱃지 · 미세 라벨             | `text-[10px]` |

### 3.3 폰트 두께

- **Regular (400)** — 대부분의 텍스트
- **Medium (500)** — 제목 · 강조 · 버튼 · 태그
- **Bold (700)** — 사용 안 함 (감성 저채도 유지)

### 3.4 자간 (letter-spacing)

| 상황               | 값        | 예시                       |
| ------------------ | --------- | -------------------------- |
| 한글 제목          | `-0.3px`  | "강릉 바다여행"            |
| 한글 본문          | 기본 (0)  | 대부분의 텍스트            |
| 영문 라벨 (대문자) | `0.5~1px` | `STOPS`, `SPENT`, `JUN 28` |
| 태그 · 뱃지        | `0.3px`   | `바다`, `산`               |

### 3.5 줄 간격 (line-height)

- 제목 (16px 이상): `1.3`
- 본문: `1.5`
- 라벨 · 부가 정보: `1.4`

## 4. 간격 (Spacing)

Tailwind 기본값 활용 (4px 배수).

| 이름      | 값   | 사용처                |
| --------- | ---- | --------------------- |
| `space-1` | 4px  | 아이콘 옆 간격        |
| `space-2` | 8px  | 태그 사이 간격        |
| `space-3` | 12px | 요소 사이 간격 (기본) |
| `space-4` | 16px | 섹션 내부 간격        |
| `space-5` | 20px | 카드 내부 패딩        |
| `space-6` | 24px | 섹션 간 간격          |
| `space-8` | 32px | 페이지 상하 여백      |

### 4.1 페이지 여백

- **좌우 패딩**: 16px (`px-4`)
- **상단 패딩**: 16px (`pt-4`)
- **하단 패딩**: 하단 탭바 높이(64px) + 16px = 80px (`pb-20`)
- **최대 너비**: 400px (모바일 위주, PC에선 가운데 정렬)

### 4.2 카드 내부 패딩

- **큰 카드** (여행 상세): `p-5` (20px)
- **중간 카드** (여행 리스트): `p-4` (16px)
- **작은 카드** (Activity 항목): `p-3` (12px)

## 12. 반응형

**모바일 우선**. PC/태블릿에서도 모바일 폭 유지.

```css
/* 앱 전체 컨테이너 */
max-width: 400px;
margin: 0 auto;

/* PC에서는 가운데 정렬, 양옆에 여백 */
@media (min-width: 768px) {
  background: #f0ebe0; /* 페이지 밖 색 */
  /* 앱 컨테이너는 여전히 400px */
}
```

이유: 여행 앱은 폰이 메인. PC에서는 폰 크기로 사용해도 문제없고, 오히려 일관성 유지에 좋음.

## 17. 안 하는 것들 (Don'ts)

- ❌ 선명한 파랑 (`bg-blue-500` 등 Tailwind 기본 색)
- ❌ 세리프 폰트
- ❌ 진한 그림자
- ❌ 그라디언트를 커버 이외에 사용
- ❌ 여러 액센트 색 혼용
- ❌ 이모지 아이콘 (Tabler로 통일)
- ❌ Bold(700) 이상의 굵은 글씨
- ❌ 형광색 · 네온 · 광택 효과
- ❌ 카드에 강한 호버 효과 (모바일 위주)
- ❌ 튀는 애니메이션

## 18. 확장 규칙

새 컴포넌트·화면 만들 때:

1. **먼저 이 문서에서 규칙 찾기**
2. 없으면 **기존 패턴을 참고**해서 만들기
3. 새 규칙이 생기면 **이 문서에 추가**
4. 색·간격을 하드코딩하지 말고 **CSS 변수 사용**

**의문이 들 때 판단 기준**:

- "감성 저채도"에 맞나?
- 청회색 톤 유지?
- 여백은 충분한가?
- 폰에서 잘 보이나?
