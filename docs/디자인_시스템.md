# 여행 관리 앱 · 디자인 시스템

> 크림 배경 + 청회색 감성 저채도 · 실용 정보 병행
> 최종 업데이트: 2026-06-30 (V2)

---

## 1. 디자인 원칙

1. **차분한 감성** — 조용하고 정돈된 톤. 과한 장식·화려한 색 배제
2. **실용 정보 병행** — 감성만이 아닌 통계·지출·평점 등 정보도 명확히
3. **모바일 우선** — 폰에서 편하게 쓸 수 있는 크기·간격
4. **일관성** — 같은 요소는 항상 같은 모양·색
5. **여백은 넉넉하게** — 밀도보다 여유

### 2.5 그라디언트 (커버 이미지 자리)

사진 없을 때 커버 이미지 placeholder로 사용.

```css
/* 기본 (일반 여행) */
--gradient-cover: linear-gradient(135deg, #6b8aa8 0%, #3a4a5c 100%);

/* 밝은 (리스트 카드) */
--gradient-cover-light: linear-gradient(135deg, #a8c0d6 0%, #6b8aa8 100%);

/* 아주 밝은 (작은 카드) */
--gradient-cover-lighter: linear-gradient(135deg, #c4d4e4 0%, #a8c0d6 100%);
```

### 2.6 색상 사용 원칙

- **한 화면에 색은 3~4가지만** (배경·텍스트·태그·강조)
- **파랑 계열은 청회색 톤**만 사용, 선명한 파랑 금지
- **갈색빛(`#B08D5C`)은 별점·즐겨찾기·하이라이트에만** 사용
- **버튼·CTA는 진한 청회색** (`#1E2A38`) 사용

## 3. 타이포그래피

### 3.1 폰트 패밀리

**한글 위주이므로 산세리프만 사용.** 세리프는 사용 안 함.

```css
--font-sans:
  "Pretendard", -apple-system, BlinkMacSystemFont, "Apple SD Gothic Neo",
  "Malgun Gothic", system-ui, sans-serif;
```

### 3.2 폰트 크기 (모바일 기준)

| 이름             | 크기 | 용도                    |
| ---------------- | ---- | ----------------------- |
| `--text-hero`    | 24px | 화면 히어로 제목        |
| `--text-title`   | 18px | 페이지 제목             |
| `--text-heading` | 16px | 섹션 제목               |
| `--text-body`    | 14px | 본문 · 리스트 항목      |
| `--text-caption` | 12px | 부가 정보 · 날짜 · 라벨 |
| `--text-micro`   | 10px | 태그 · 뱃지 · 미세 라벨 |

### 3.3 폰트 두께

- **Regular (400)** — 대부분의 텍스트
- **Medium (500)** — 제목 · 강조 · 버튼 · 태그
- **Bold (700)** — 사용 안 함

### 3.4 자간 (letter-spacing)

| 상황               | 값        | 예시              |
| ------------------ | --------- | ----------------- |
| 한글 제목          | `-0.3px`  | "강릉 바다여행"   |
| 한글 본문          | 기본 (0)  | 대부분의 텍스트   |
| 영문 라벨 (대문자) | `0.5~1px` | `STOPS`, `JUN 28` |
| 태그 · 뱃지        | `0.3px`   | `바다`, `산`      |

## 4. 간격 (Spacing)

Tailwind 기본값 활용 (4px 배수).

### 4.1 페이지 여백

- **좌우 패딩**: 16px (`px-4`)
- **상단 패딩**: 16px (`pt-4`)
- **하단 패딩**: 하단 탭바 높이(64px) + 16px = 80px (`pb-20`)
- **최대 너비**: 400px

### 4.2 카드 내부 패딩

- **큰 카드** (여행 상세): `p-5` (20px)
- **중간 카드** (여행 리스트): `p-4` (16px)
- **작은 카드** (Activity, Day 카드): `p-3` (12px)

### 7.7 별점 및 컨디션

**별점 (Activity·Trip·Day 컨디션)**

- **문자**: `◆` (다이아몬드)
- **색**: 채워진 것 `#B08D5C`, 빈 것 `#E8E4D8`
- **개수**: 5개 (별점·컨디션 모두 동일)
- **크기**: 부모 폰트 크기 그대로

**표기 예시**

- 5점: `◆◆◆◆◆` (다 채움)
- 4점: `◆◆◆◆◇` (마지막 하나 빈 색)
- 컨디션 4/5: `◆◆◆◆◇`

### 7.8 즐겨찾기 아이콘

**"다시 가고 싶은 곳" 표시**

- **아이콘**: 별 (`ti-star-filled`) 또는 하트 (`ti-heart-filled`)
- **활성 색**: `#B08D5C`
- **비활성 색**: `#A8B4C4`
- **크기**: 16~18px

두 아이콘 중 하나 선택 사용 (개인 취향, 코드에서 통일).

## 12. 페이지별 레이아웃

### 12.1 홈 (여행 리스트)

```
- 상단 상태바
- 헤더: 제목 "여행" + SIX MEMORIES 서브텍스트 + 우측 돋보기
- 필터 칩: 전체 · 국내 · 해외 · 연도
- 여행 카드 리스트 (세로 스크롤)
- 하단 탭바 + FAB
```

### 12.2 검색 페이지

```
- 상단 상태바
- 검색 헤더: 뒤로가기 + 검색창 (rounded-full)
- 필터 칩: 타입(식당·방문지) · 평점 · 재방문
- 결과 개수 표시
- 결과 카드 리스트
```

### 12.3 상세 페이지 (Day 카드 스택)

```
- 상단 상태바 (다크)
- 히어로 커버 (110px, 그라디언트 또는 사진)
  - 좌측: 뒤로가기 / 우측: 사진 추가 아이콘
  - 하단 오버레이: 태그 + 나라뱃지 + 제목 + 경로·날짜·동행자
- 통계 로우: 일정 · 지출 · 평점
- Day 카드 스택 (일자별 반복)
```

### 12.4 Day 카드 구조

```
- Day 헤더: DAY N · 날짜 (왼쪽) + 컨디션 (오른쪽)
- 자동 요약: 방문 N곳 · 지출 · 사진 N장
- 하이라이트 박스 (있을 때만): --bg-highlight
- 구분선
- Activity 미리보기 (3개까지):
  - 시간 · 아이콘 · 이름
```

### 12.5 식당 페이지

```
- 상단 상태바
- 헤더: 뒤로가기 + 제목 "식당 정보" + 공유
- 식당 정보 섹션:
  - 상호명 (18px, 500)
  - 주소 (아이콘 + 텍스트)
  - 태그 (음식 종류)
  - 구분선
  - 평균 평점 (◆ 4.5 · 방문 N회 평균)
  - 즐겨찾기 표시 (별/하트 아이콘 + "다시 가고 싶은 곳")
- 방문 이력 헤더: "방문 이력 · N회"
- 방문 이력 카드 스택 (최근순)
```

### 12.6 요약본 페이지 (폴라로이드)


## 18. 안 하는 것들 (Don'ts)

- 선명한 파랑 (`bg-blue-500` 등)
- 세리프 폰트
- 진한 그림자
- 그라디언트를 커버 이외에 사용
- 여러 액센트 색 혼용
- 이모지 아이콘 (Tabler로 통일)
- Bold(700) 이상
- 형광색 · 네온 · 광택
- 카드에 강한 호버 효과
- 튀는 애니메이션

## 19. 확장 규칙

새 컴포넌트·화면 만들 때:

1. **먼저 이 문서에서 규칙 찾기**
2. 없으면 **기존 패턴을 참고**해서 만들기
3. 새 규칙이 생기면 **이 문서에 추가**
4. 색·간격을 하드코딩하지 말고 **CSS 변수 사용**

**판단 기준**:

- "감성 저채도"에 맞나?
- 청회색 톤 유지?
- 여백은 충분한가?
- 폰에서 잘 보이나?
