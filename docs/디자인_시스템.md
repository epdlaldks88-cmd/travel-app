# 여행 관리 앱 · 디자인 시스템

> 감성 저채도 · 실용 정보 병행 · **테마 시스템 지원 (8개 프리셋)**
> 최종 업데이트: 2026-07-03 (V3)

---

## 1. 디자인 원칙

1. **차분한 감성** — 조용하고 정돈된 톤. 과한 장식·화려한 색 배제
2. **실용 정보 병행** — 감성만이 아닌 통계·지출·평점 등 정보도 명확히
3. **모바일 우선** — 폰에서 편하게 쓸 수 있는 크기·간격
4. **일관성** — 같은 요소는 항상 같은 모양·색
5. **여백은 넉넉하게** — 밀도보다 여유
6. **테마 대응** — 하드코딩 대신 semantic 토큰 사용, 8개 테마 자동 대응

## 2. 색상 시스템 (테마 대응)

| `--color-surface`       | 카드 · 폼 · 시트 배경                       |
| `--color-surface-alt`   | 살짝 다른 톤 (input 배경, hover, 태그 배경) |

| `--color-accent`        | 주요 강조 (FAB, 선택 상태, 링크)            |

- 채운 다이아: `#D4A017`
- 빈 다이아: 같은 색 + opacity 0.15

정의 위치: `src/theme/tokens.js` 의 `rating.color`

### 2.6 색상 사용 원칙

- **하드코딩 hex 사용 금지** — 모든 색은 semantic 토큰(`bg-surface`, `text-text-muted` 등)으로 표현
- **한 화면에 강조 색은 accent 하나만** — danger/success는 상태 표현용, 강조로 남용 X
- **오버레이(이미지 위)**는 예외 — `bg-white/90`, `bg-black/30` 등 하드코딩 유지 가능
- **테마 무관 커스텀 색이 필요하면** — 새 semantic 토큰을 추가 (예: `--color-hero-from`)

## 7. UI 컴포넌트

**`src/components/ui/` 에 있는 12개 재사용 컴포넌트를 우선 사용.** 하드코딩 HTML 요소보다 이 컴포넌트들을 조합하는 게 원칙.

### 7.1 Card

**용도**: surface 컨테이너.

```jsx
<Card padding="md" onClick={handler}>
  <p>내용</p>
</Card>
```

Props:

- `padding`: `none` / `sm` / `md` (기본) / `lg`
- `onClick`: 있으면 hover · active 스타일 자동 적용
- `as`: 렌더링 태그 (기본 `div`)

### 7.2 Button

```jsx
<Button variant="primary" size="md" leftIcon={<IconPlus size={18} />}>
  여행 추가
</Button>
```

Props:

- `variant`: `primary` (accent 배경) / `secondary` (surface + border) / `ghost` / `danger`
- `size`: `sm` (h-8) / `md` (h-10, 기본) / `lg` (h-12)
- `leftIcon` / `rightIcon`: 아이콘 JSX
- `loading`: 스피너 + disabled
- `fullWidth`: `w-full`

### 7.3 Chip

**용도**: 카테고리 태그, 필터, 다중/단일 선택.

```jsx
<Chip variant="selected" onClick={toggle} icon={<IconCar size={14} />}>
  자동차
</Chip>
```

Props:

- `variant`: `default` (surface-alt 배경) / `selected` (accent 배경) / `outline`
- `icon`: 좌측 아이콘
- `onClick`: 있으면 button 태그로 렌더

### 7.4 Input · Textarea · Select

**공통**: `error`, `size`, forwardRef 지원.

```jsx
<Input value={x} onChange={e => setX(e.target.value)} placeholder="..." />
<Textarea rows={2} className="resize-none" />
<Select options={[{value, label}]} placeholder="선택" />
```

Focus 상태 (border-strong)는 자동 처리. 인라인 `onFocus/onBlur` 사용 금지.

### 7.5 Label

```jsx
<Label htmlFor="name" required>
  이름
</Label>
```

Props: `htmlFor`, `required` (별표 표시).

### 7.6 Badge

**용도**: 카운트, 상태 뱃지.

```jsx
<Badge variant="danger" size="sm">3</Badge>
<Badge dot variant="success" />
```

Props: `variant` (`default`/`accent`/`danger`/`success`/`muted`), `size` (`sm`/`md`), `dot`.

### 7.7 Section

**용도**: 페이지 내부 그룹핑 (제목 + 액션 슬롯).

```jsx
<Section
  title="테마"
  description="선호 색상 선택"
  action={<Button>더보기</Button>}
>
  {children}
</Section>
```

### 7.8 PageHeader

**용도**: 페이지 최상단 헤더 (뒤로가기 + 액션).

```jsx
<PageHeader
  title="여행 상세"
  back
  onBack={customBack}
  action={<Button>편집</Button>}
  sticky
/>
```

### 7.9 Rating (⭐ 핵심 컴포넌트)

**◆ 다이아몬드 평점. 0.5 단위 지원.**

```jsx
<Rating value={3.5} onChange={setRating} size="lg" showValue />
<Rating value={4.5} readonly size="sm" />
```

Props:

- `value`: 0~max, 0.5 단위
- `max`: 기본 5
- `readonly`: true면 표시만
- `size`: `sm` (16px) / `md` (20px, 기본) / `lg` (26px)
- `showValue`: 우측에 "3.5" 숫자 표시
- 편집 모드: 각 다이아를 좌/우 반쪽으로 나눠 클릭 (좌 = +0.5, 우 = +1.0)
- 같은 값 재클릭 시 0으로 해제

### 7.10 Switch

**iOS 스타일 토글.**

```jsx
<Switch checked={autoDetect} onChange={setAutoDetect} aria-label="자동 감지" />
```

Props: `checked`, `onChange(newValue)`, `disabled`, `size` (`sm`/`md`).

### 7.11 서브컴포넌트 패턴 (파일 내부)

파일 안에서 3번 이상 반복되는 UI는 서브컴포넌트로 뽑기. 예:

- `ActivityItem` 안의 `MiniBadge` (짧은 텍스트 뱃지)
- `TripDetailPage` 안의 `StatCell` (통계 로우 셀)
- `ProfilePage` 안의 `ThemeCard`, `PaletteCircle`, `DataMenuItem`

- 위치: 탭바 중앙 위 -16px 겹침 (`-mt-4`)

**⚠️ 크림 테마 FAB는 원본 `#1E2A38` 이 아닌 `#B08D5C` (갈색빛 accent).** 원본 감성을 유지하고 싶으면 크림 테마의 `accent` 값을 조정하거나 별도 `--color-fab` 토큰 추가.

## 18. 안 하는 것들 (Don'ts)

- **하드코딩된 hex 색** (오버레이 뱃지, 히어로 그라데이션 대체 이미지 자리 등 명확한 예외만 허용)
- 인라인 `style={{ color, background, ... }}` (반드시 필요한 arbitrary 값만 예외)
- `onFocus/onBlur`로 인풋 border 색 변경 → Input 컴포넌트가 자동 처리
- 커스텀 별점 렌더 → Rating 컴포넌트 사용
- 이모지 아이콘 (📍 🚙 ✕ 등) → Tabler로 통일
- 선명한 파랑 (`bg-blue-500` 등)
- 세리프 폰트
- 진한 그림자
- `Bold(700)` 이상 (Semibold까지만)
- 형광색 · 네온 · 광택
- 튀는 애니메이션

## 19. 확장 규칙

새 컴포넌트 · 화면 만들 때:

1. **먼저 `src/components/ui/` 에서 재사용 가능한 컴포넌트 확인**
2. 없으면 **기존 도메인 컴포넌트에서 유사 패턴 참고**
3. 3번 이상 반복되는 UI는 서브컴포넌트로 뽑기 (파일 내부 or `ui/`)
4. 새 semantic 토큰이 필요하면 **`tokens.js` / `themes.js` 에 12개 토큰 확장**
5. `index.css` 의 `@theme inline` 에 매핑 추가
6. **하드코딩 hex 사용 금지** — 예외는 오버레이/그라데이션 대체 이미지 자리만
7. 새 규칙 생기면 **이 문서에 추가**

**판단 기준**:

- 감성 저채도 유지?
- 테마 전환 시 자연스러운가?
- 여백은 충분한가?
- 폰에서 잘 보이나?
- 접근성 (터치 타겟, 대비, 라벨)?

## 20. 테마 시스템 참조

### 20.1 파일 위치

```
src/theme/
├── tokens.js           # 공통 토큰 (spacing/radius/shadow/typography/transition/rating)
├── themes.js           # 8개 테마 프리셋 + helper
├── ThemeContext.jsx    # Provider (CSS 변수 주입, localStorage, 시스템 감지)
└── useTheme.js         # 소비용 훅
```

### 20.2 ThemeProvider 사용

`src/main.jsx` 에서 `<App />` 을 감싸면 앱 전체가 테마 대응.

```jsx
import { ThemeProvider } from "./theme/ThemeContext";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <ThemeProvider>
      <App />
    </ThemeProvider>
  </StrictMode>,
);
```

### 20.3 useTheme 훅

```jsx
import { useTheme } from "@/theme/useTheme";

const {
  theme,
  themeKey,
  mode,
  autoDetect,
  setTheme,
  setAutoDetect,
  allThemes,
```

### 20.4 localStorage 스키마

```
key:   travel-app:theme
value: { key: 'cream', autoDetect: true }
```

수동으로 테마 선택 시 `autoDetect` 자동 `false`. 마이페이지에서 언제든 다시 활성화 가능.

### 20.5 새 테마 추가 방법

1. `themes.js` 의 `themes` 객체에 새 프리셋 추가 (14개 색상: 12 semantic + heroFrom + heroTo)

3. 추가 코드 수정 불필요
